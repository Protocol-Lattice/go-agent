Why GO-ADK?

Modern agents demand more than a single model call. GO-ADK orchestrates models, tools, memory, and multi-agent workflows so you can focus on domain-specific logic instead of plumbing.

Orchestrated Runtime

Configure coordinator models, tool catalogs, sub-agent registries, and memory stores using functional options. Sessions are thread-safe and deterministic.

Modular Kit

Compose ready-made modules for models, memory, tools, and runtime configuration. Swap components without rewriting your application.

Tooling Ecosystem

Implement the agent.Tool interface once and register it everywhere. Built-in examples cover calculators, clocks, echo responders, and more.

Advanced Memory

Leverage importance scoring, recency weighting, maximal marginal relevance, and pluggable vector stores like PostgreSQL/pgvector or Qdrant.

Architecture Overview

The kit exposes a layered design so you can start with a simple runtime and grow into modular deployments. Each package focuses on a single responsibility, empowering you to extend or replace components as your product evolves.

  • pkg/runtime: Session management, orchestration, UTCP tool calling.
  • pkg/agent: Coordinator logic, tool routing, and sub-agent delegation.
  • pkg/memory: Retrieval augmented memory with configurable heuristics.
  • pkg/models: Gemini, Anthropic, Ollama, and dummy adapters behind a unified interface.
  • pkg/subagents: Specialist personas wired through the ToolCatalog.
  • pkg/tools: Built-in tools ready to register.

Quick Start

Install

git clone https://github.com/Raezil/go-agent-development-kit.git
cd go-agent-development-kit
go mod download

Configure

export GOOGLE_API_KEY="your-gemini-api-key"
export DATABASE_URL="postgres://admin:admin@localhost:5432/ragdb?sslmode=disable"
export ADK_EMBED_PROVIDER="gemini"

Run a Demo

go run ./cmd/demo --dsn "$DATABASE_URL"

Or try the zero-config sample with go run ./cmd/quickstart.

Capabilities

Memory Engine

Weighted retrieval, clustering, summarisation, and pruning keep context relevant without sacrificing recall.

  • Configurable similarity vs. recency weightings
  • Source boosting for critical signals
  • MMR and summarisation pipelines

Tooling & Sub-Agents

Register any number of tools and specialist personas. UTCP keeps invocation provider-agnostic.

  • Declarative tool catalogs
  • Sub-agent directories
  • Runtime prompts for orchestration

Deployment Ready

Integrates with PostgreSQL/pgvector, ships with deterministic dummy models, and follows Go best practices for testing.

  • In-memory mode for local prototyping
  • Schema bootstrapping via runtime options
  • Interfaces for plugging in custom providers
